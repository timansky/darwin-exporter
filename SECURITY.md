# Security Policy

## Supported Versions

For this project stage, security fixes are applied to `main` only.

| Version              | Supported |
| -------------------- | --------- |
| `main`               | yes       |
| tags / older commits | no        |

## Reporting a Vulnerability

Please do not open a public issue for a security vulnerability.

Use one of the following private channels:

1. GitHub Security Advisory ("Report a vulnerability" in the repository Security tab).
2. Email: `timansky@neko.kz`

Include:

1. Affected version/commit (`git rev-parse --short HEAD` if possible).
2. Reproduction steps and expected vs actual behavior.
3. Impact assessment (confidentiality, integrity, availability).
4. Any available proof-of-concept.

## Response Targets

1. Initial acknowledgement: within 72 hours.
2. Triage update: within 7 days.
3. Fix timeline: depends on severity and reproducibility.

## Disclosure

Please keep reports private until a fix is released and maintainers confirm disclosure.
